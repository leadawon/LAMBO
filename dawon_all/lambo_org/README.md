# lambo_org — retrieval baseline (sibling of `lambo`)

`lambo_org` is a cleaner baseline variant of the original `lambo` pipeline.
It **removes** our custom "anchor summary generation" contribution and
**replaces** it with a plain chunk-embedding + cosine-similarity retrieval
baseline. The original `lambo/` package is untouched.

## What changed

| Stage | Original (`lambo`) | This baseline (`lambo_org`) |
| --- | --- | --- |
| Anchoring | `AnchorAgent` — LLM groups units into semantic anchors with `anchor_title` / `summary` / `key_entities` | `RetrievalAnchorAgent` — splits docs into plain chunks, embeds them, ranks by cosine similarity against the query |
| Per-doc evidence | `DocRefineAgent` — iterative LLM think→search→info loop over anchor summaries | `RetrievalDocRefineAgent` — single pass, picks top-K chunks directly from the similarity ranking |
| Cross-doc compose | `GlobalComposer` (unchanged) | `GlobalComposer` (reused from `lambo`) |
| Final answer | `Generator` (unchanged) | `Generator` (reused from `lambo`) |

The pseudo-anchor schema is preserved so downstream `GlobalComposer` /
`Generator` work without modification. Each chunk becomes an "anchor" whose
`anchor_title` / `summary` are derived from raw text (NOT LLM output) and
whose `key_entities` is empty.

## Retrieval behavior

1. Each document is chunked by the same paragraph-based splitter used in
   the original anchor agent (copied locally so `lambo_org` is standalone).
2. Each chunk is embedded with a configurable `sentence-transformers`
   model (default: `all-MiniLM-L6-v2`).
3. The query string is `question + "\n" + instruction`, embedded once per
   record.
4. Chunks are ranked by cosine similarity.
5. `retrieval_scope`:
   - `per_document` (default): keep top-K chunks per document.
   - `global`: keep top-K chunks across all documents.

## 99-example Loong experiment

Follows dawonv7/dawonv8 conventions: a deterministic balanced SET1 subset
of 99 Loong rows (33 each from `paper`, `legal`, `financial`), held in
`data/loong_set1_balanced99_indices.json`, generated by
`prepare_exper99_subset.py`.

### Commands

Single entrypoint (mirrors `dawonv7/run_exper99.sh`): builds the 99-sample
subset if needed, then runs inference.

```bash
# Inference on the 99 subset
bash dawon_all/lambo_org/run_exper99.sh

# Evaluation (writes metrics.json / scores.json / summary.txt under <OUTPUT_DIR>/reports)
python dawon_all/lambo_org/run_eval.py --output_dir dawon_all/lambo_org/logs/lambo_org_set1_exper99
```

Common env vars:

| Var | Default | Notes |
| --- | --- | --- |
| `LAMBO_ORG_PYTHON` | `/workspace/venvs/ragteamvenv/bin/python` | Python interpreter |
| `LAMBO_ORG_INPUT_PATH` | `/workspace/StructRAG/loong/Loong/data/loong_process.jsonl` | Full Loong JSONL |
| `LAMBO_ORG_OUTPUT_DIR` | `dawon_all/lambo_org/logs/lambo_org_set1_exper99` | Where predictions/reports go |
| `LAMBO_ORG_EMBEDDING_MODEL` | `sentence-transformers/all-MiniLM-L6-v2` | Any SBERT-compatible model |
| `LAMBO_ORG_TOP_K_PER_DOC` | `6` | chunks kept per doc |
| `LAMBO_ORG_TOP_K_GLOBAL` | `24` | chunks kept globally (scope=global) |
| `LAMBO_ORG_RETRIEVAL_SCOPE` | `per_document` | `per_document` or `global` |
| `LAMBO_ORG_BACKEND` | `local` | `local` (Qwen) or `gemini` |
| `LAMBO_ORG_SKIP_JUDGE` | `0` | Set to `1` to skip the LLM-judge pass |

## Where results are saved

```
<OUTPUT_DIR>/
  manifest.json                 # the 99 selected indices + sample_ids
  lambo_predictions.jsonl       # raw predictions (one per sample)
  samples/<sample_id>/
      anchors.json              # pseudo-anchors + retrieval ranks (no LLM summaries)
      doc_refine_DOC*.json      # top-K chunks per doc
      composed.json             # GlobalComposer output
      generator.json            # Generator output
  reports/
      metrics.json              # structured + judge metrics
      scores.json               # flat numeric scores
      summary.txt               # human-readable summary
      structured_eval.json
      llm_judge.json
      errors.json
```

## Assumptions / caveats / TODOs

- Requires `sentence-transformers`; `lambo_org.embeddings` raises a clear
  install hint if it is missing.
- `GlobalComposer` and `Generator` still call the LLM — only the anchor
  summary generation and the iterative per-doc refinement were removed.
- No smoke run was executed in this patch (embedding model download +
  Qwen backend not guaranteed in the sandbox). The Python import path has
  been validated and the entrypoints match dawonv7/dawonv8 CLI style.
- The evaluation scripts reuse `lambo.eval.structured_eval` and
  `lambo.eval.llm_judge` unchanged for fair comparability.
